# 2026-05-15 管理端更改預約時間

## 需求摘要

新增管理員手動更改學員預約課程時間的設計與實作，並確保日誌、出缺勤、票券 audit 與低成本公開同步一起連動。

## 變更檔案

- `github_pages/admin.html`

## 行為變更

- 學員詳細頁的未來預約列新增「改時間」按鈕。
- 管理員可將預約移動到同課程類型、未來、未額滿、且該學員尚未預約的其他時段。
- 同課程類型改時間不補堂、不扣堂。
- 移動時同步更新：
  - `slots[].bookings`
  - `students[].scheduledBookings`
  - `classes` 出缺勤資料
  - `course_logs` 課程日誌
  - `tickets.log` / `tickets.logs` 的 `booking_move` audit 紀錄
- 若原課程已開始或已有非 `pending` 出缺勤狀態，會阻擋改時間，避免破壞歷史紀錄。
- 多人共用同一筆 `course_logs` 時，只拆出被移動的學員到新時段，其他學員保留在原日誌。

## Firestore 讀寫影響

- 未新增學生端 Firestore 讀取。
- 未新增學生端 `getDocs()`、大量 `onSnapshot()`，也未讓學生端讀 `/data/students` 或 `/data/tickets`。
- 新功能只在管理端使用既有 `saveSlots()`、`saveStudents()`、`saveClasses()`、`saveLogs()`、`saveTickets()` 寫回 legacy data。
- `saveSlots()` / `saveStudents()` 仍透過既有 `lowCostScheduleSync()` debounce 更新 `public_booking/state` 與必要的 `student_lookup/{hash}`。
- 學生端讀寫預算維持原本低成本模式。

## 驗證

- 使用 Node 隔離抽取新增函式做情境測試：
  - 單人日誌：預約從舊時段移到新時段，日誌保留內容並更新 `slotId/date/time`。
  - 多人日誌：原日誌保留未移動學員，移動學員拆出新日誌並保留 `note/alerts`。
  - 票券 `left/used` 不變，但新增 `booking_move` audit log。
- 使用 `rg` 檢查新增改動沒有在學生端加入新的 `getDocs()` 或大量監聽路徑。

## 後續風險

- 目前第一版只支援同課程類型改時間；跨課程類型需要另外設計票券退扣與方案類型校驗。
- `docs/ssot/` 在此工作版不存在，因此本次未更新 SSOT 文件。
