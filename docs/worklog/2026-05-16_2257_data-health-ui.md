# 2026-05-16 22:57 資料健康檢查 UI 優化

## 需求摘要

管理員按下「資料一致性檢查」後，需要更清楚知道下一步要做什麼。原本報告會列出異常，但管理員不容易判斷哪些要先處理、哪些可以按工具修復、哪些應人工確認。

## 行為變更

- 資料健康檢查摘要改為顯示「嚴重、可修復、人工確認、學員」四個指標。
- 新增「建議處理順序」，讓管理員知道該先處理嚴重項目，再處理可修復項目，最後重新檢查。
- 異常卡片分組為「嚴重項目、可修復項目、人工確認項目」。
- 可修復項目會在卡片內顯示對應按鈕，例如重建學員查詢、清除孤兒日誌成員、查看票券重算報告。
