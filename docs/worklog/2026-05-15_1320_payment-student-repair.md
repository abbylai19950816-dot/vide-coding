# 2026-05-15 收款後補建學員資料

## 需求摘要

使用者回報柯宇恆購買一對二體驗課，管理員標記收款後沒有寫入學員資料。

## 變更檔案

- `active/gyrobooking_current/github_pages/admin.html`
- `admin.html`

## 行為變更

- `createTicketFromPayment()` 在標記已收款並開票券前，若付款紀錄找不到對應學員，會用付款紀錄上的姓名、手機、備註補建學員資料。
- 補建學員後，會同步修正該筆付款紀錄的 `studentId`、`studentName`、`phone`、`color`。
- 若付款手機已被另一位既有學員使用，會阻擋自動補建並提示管理員先確認付款紀錄，避免把不同學員合併錯。

## Firestore 讀寫影響

- 學員端無變更。
- 管理員端只有在「已收款付款紀錄找不到學員」時，才額外寫入 `/data/students` 與修正 `/data/payments`。
- 正常已有學員的收款流程不增加讀寫。

## 驗證

- 檢查 `createTicketFromPayment()` 尋找學員、補建學員、開票券流程。
- 執行管理員頁 script 語法檢查。
- 執行低成本關鍵字檢查，確認學員端無新增讀寫路徑。

## 後續風險

- 已經標記為已收款但當時沒有建立學員的舊紀錄，不會自動重跑；可在修正上線後將該收費改回待收款再標記已收款，或手動新增學員與方案。
