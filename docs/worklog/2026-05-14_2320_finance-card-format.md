# 2026-05-14 收費卡片格式調整

## 需求摘要

管理員希望收費頁的學員辨識資訊改成：

- 第一行：姓名 手機
- 第二行：課程方案名稱

姓名需為黑色粗體；手機與課程方案名稱維持原本較淡、較小的視覺層級。收費卡片上的待收款或待確認類標籤可移除，避免畫面資訊過重。

## 變更檔案

- `active/gyrobooking_current/github_pages/admin.html`
- `admin.html`

## 行為變更

- 收費紀錄卡片改為姓名與手機同列、課程方案獨立下一列。
- 待收款卡片改為同樣的姓名、手機、課程方案格式。
- 收入報表的學員統計也使用姓名與手機同列，筆數保留在下一列。
- 收費紀錄卡片移除 `已收` / `待收` badge。
- 待收款卡片移除日期天數提示，只保留金額與操作按鈕。

## Firestore 讀寫影響

- 無新增 Firestore 讀取、寫入或 listener。
- 學員端無變更。
- 此次僅調整管理員端既有付款資料的渲染格式。

## 驗證

- 檢查收費紀錄、待收款、收入報表學員統計的 HTML template。
- 執行管理員頁 script 語法檢查。
- 執行低成本關鍵字檢查，確認沒有新增讀寫路徑。

## 後續風險

- 若付款紀錄與學員資料都沒有手機，仍會顯示 `未留手機`。
