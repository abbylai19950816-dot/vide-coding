# 學員預約與管理員管理 E2E 測試紀錄

## 請求摘要

使用者允許建立一組可刪除測試資料，例如 `Codex測試學員 / 0900000000`，用來測試學員預約全流程與管理員管理全流程。

## 測試方式

- 使用 Firebase REST API 建立帶有 `CodexE2E_` 標記的測試資料。
- 只新增測試學員、測試收費、測試票券、測試時段、測試出缺勤資料。
- 不碰未帶測試標記的真實資料。
- 測完立即依 `testRunId`、測試姓名、測試時段 ID 清除資料。

## 通過項目

- 讀取目前 Firebase 資料摘要。
- 學員購課送出：建立測試學員與 `unpaid` 收費紀錄。
- 管理員標記收款：將收費改為 `paid` 並建立票券。
- 管理員同步學生查詢資料：建立 `student_lookup/{hash}`，剩餘堂數正確。
- 學員查詢剩餘堂數與公開課表：讀取 `student_lookup/{hash}` 與 `public_booking/state`。
- 學員單次預約：扣 1 堂、寫入 slot booking、寫入學生 `scheduledBookings`。
- 學員循環預約：依同課程、同時間、未滿班、未過期條件選取後續時段並扣堂。
- 管理員取消預約：移除 slot booking、移除學生未來預約、票券補回 1 堂。
- 管理員出缺勤：將測試課程出缺勤由 `pending` 改為 `present`。
- 清理測試資料：測試資料已刪除。
